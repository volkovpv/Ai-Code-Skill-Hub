# The convention checker over-reports env access without a file path

Running `scripts/check_conventions.py` on a configuration-layer file decides
whether `process.env` is permitted from the file **path**. Piping the file
through stdin (label `<stdin>`) removes that context, so legitimate env reads
in the config layer are reported as `TS-ENV`.

Reproduction: `python scripts/check_conventions.py data/fixtures/settings.config.ts`
finds nothing (the `.config.ts` name marks the config layer), while
`python scripts/check_conventions.py < data/fixtures/settings.config.ts`
reports two `TS-ENV` findings.

Suggested consequence (pending review): pass real file paths to the checker
(do not pipe config-layer code through stdin), or read a `TS-ENV` finding in
light of the file's actual layer. If accepted, this may later be promoted to
`knowledge/pitfalls.md` as a separate reviewed change.
