## Context (input)

Routine use of the installed `typescript-coding` skill (`install_mode: runtime`,
`.agent-skills.lock.yaml`): an agent following the skill's own calibration pointers into
`../data/fixtures/*`, and a reader of the skill's observation registry following its single linked
entry.

Two shipped directories name content the runtime install does not carry:

```
.agents/skills/typescript-coding/
├── data/
│   ├── README.md          ← names `fixtures/` as an existing sibling of `examples/`
│   └── examples/          ← the ONLY data directory actually shipped
└── observations/
    ├── INDEX.md           ← links `candidates/OBS-20260713-001.md`
    └── accepted/          ← the ONLY observation directory actually shipped
```

## Expected behavior

A pointer inside shipped skill content resolves in the tree the consumer actually receives. The
observation registry is the sharper case: `observations/INDEX.md` **states the rule itself** —
«candidates and rejected observations are development-only content and are [not shipped]» — and then
links one. A document that documents its own packaging boundary and then crosses it is worse than a
plain broken link: it teaches the reader that the boundary is not real.

## Actual behavior

First upstream failure, as data: **4 references to `../data/fixtures/*` across 3 shipped files** and
**1 registry link into the unshipped `candidates/`** resolve to nothing.

| Shipped file | Line(s) | Target | Class |
|---|---|---|---|
| `knowledge/patterns.md` | 13, 21, 39 | `../data/fixtures/clean_sample.ts` | packaged-out fixtures |
| `knowledge/pitfalls.md` | 17 | `../data/fixtures/masked_literals.ts` | packaged-out fixtures |
| `references/typing-and-style.md` | 91 | `../data/fixtures/clean_sample.ts` | packaged-out fixtures |
| `observations/INDEX.md` | 34 | `candidates/OBS-20260713-001.md` | self-contradicting registry link |

`observations/accepted/OBS-20260715-001.md` additionally cites
`data/fixtures/justified_rule_disable.ts` as its own reproduction fixture (lines 7, 46) — an accepted
observation whose evidence is unreachable in the shipped tree, so the record cannot be re-verified by
the consumer it ships to.

## Universality check

Independent of any local norm, TypeScript version, framework or project layout: the defect lives
entirely inside the skill's own shipped tree and reproduces on an empty target directory. It touches
no project threshold and no local rule, so SFL-INV-15 does not apply. The fixture half is
byte-for-byte the same defect that ships in `python-coding` ([[OBS-20260721-001]]) — two independent
skills, one packaging-convention gap — which is why the fix belongs to the Hub's packaging/validation
convention and not to either skill's prose alone.

SFL-INV-08 threshold: met — 5 independent occurrences across 4 files in this skill, a second affected
skill, **and** a deterministic, project-independent minimal reproduction.
