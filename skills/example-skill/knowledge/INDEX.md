# Knowledge index — example-skill

Verified, generalizable knowledge for drafting commit messages. Read a file
only when its trigger matches the current task; do not preload everything.

| File | Read when |
|------|-----------|
| [patterns.md](patterns.md) | composing the final message: choosing type, scope and subject |
| [pitfalls.md](pitfalls.md) | the script's suggestion looks wrong, or the diff is mixed/ambiguous |

Rules for adding knowledge:

- only verified, generalizable statements with an explicit applicability scope;
- every entry links to its evidence (test, fixture, or accepted observation);
- do not duplicate the main workflow from SKILL.md;
- files longer than 100 lines must start with a short table of contents.
