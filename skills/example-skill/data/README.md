# Data contract — example-skill

- **Purpose**: reproducible inputs for the commit-message helper.
  `fixtures/` feeds the automated tests of this skill; `examples/` shows a
  minimal input/output pair to calibrate expectations.
- **Source / generation**: hand-written synthetic diffs; no real repository
  content. Regenerate by editing the files directly — they are small enough
  that scripted generation would add complexity without benefit.
- **License**: MIT (same as the skill; see ORIGIN.yaml).
- **Format**: unified diff (`git diff` output) in `.diff` files; expected
  script output in matching `.expected` text files.
- **Allowed use**: input for `scripts/example.py` in tests and demos; the
  data never overrides the agent's own judgement about a real diff.
- **PII / secrets**: none — all paths, identifiers and content are invented;
  the library's secret scan runs over this directory on every validation.
- **Update procedure**: edit → `skillctl validate example-skill` →
  `skillctl test example-skill` → bump `version` in skills.yaml.
- **Size limits**: each file stays far below the default
  `content_policy.max_tracked_file_bytes` (256 KiB); anything bigger must be
  generated by a script instead of being committed.

Layout convention:

- `fixtures/` — test-only inputs; **not** installed in runtime mode;
- `examples/` — minimal input/output pairs shipped to consumers.
