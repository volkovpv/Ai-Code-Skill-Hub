# Deletion-heavy fix diffs are classified as `refactor`

Running `scripts/example.py` on a bug fix that removes a faulty workaround
(5 lines removed, 1 added, no fix keywords in added lines) yields
`refactor(src): update cache.py`, while the correct type is `fix`.

Reproduction: feed `data/fixtures/mixed_change.diff` to the script; the
dedicated test pins this behaviour so any classifier change is visible.

Consequence: promoted to `knowledge/pitfalls.md` («Deletion-heavy fixes are
misclassified as refactor») — the agent must override the type when the
intent of the change is a fix.
