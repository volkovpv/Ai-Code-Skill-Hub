# Subject language should follow the repository history

In repositories whose commit history is written in Russian, an English
subject produced by this skill looks inconsistent in `git log`. SKILL.md
already says «keep the subject in English unless the repository history uses
another language», but there is no evidence yet on how reliably agents detect
the dominant language of the history.

Needed before approval: a reproducible check (e.g. a fixture with a synthetic
`git log` sample) demonstrating detection of the dominant language.
